WeekFoods: tu planificador semanal de menús

La planificación de comidas puede ser un desafío, consumiendo tiempo y esfuerzo. WeekFoods simplifica este proceso, ayudándote a comer de forma saludable, variada y ajustada a tu presupuesto, todo ello desde una plataforma intuitiva y eficiente.

Cómo funciona

1. Gestión de tus recetas

En WeekFoods, tienes el control total sobre tu colección de recetas. Puedes añadir tus platos favoritos, tanto para comidas como para cenas, detallando sus ingredientes, calorías y el coste estimado. Esta biblioteca personal es la base de tus menús futuros.

Beneficio: Construye tu propia biblioteca de platos favoritos que el sistema usará para elaborar tus menús, garantizando que siempre disfrutes de tus comidas preferidas.

2. Menú semanal

Olvídate de pensar qué cocinar cada día. Introduce tus preferencias de calorías máximas diarias y tu presupuesto máximo, y WeekFoods generará automáticamente un menú de 10 recetas (5 comidas y 5 cenas) para tu semana laboral, seleccionándolas de tu colección personal.

Beneficio: Ahorra tiempo y esfuerzo en la planificación, obteniendo un menú que se ajusta perfectamente a tus objetivos nutricionales y económicos.

3. Personalización y flexibilidad

¿El menú generado no es exactamente lo que buscas? No hay problema. WeekFoods te da el control total. Puedes cambiar recetas específicas, seleccionar manualmente cualquier plato de tu lista para un día concreto, o incluso eliminar todo el menú para empezar de cero.

Beneficio: Tienes el control total para ajustar el menú a tus preferencias y disponibilidad, garantizando que siempre disfrutes de lo que comes.

4. Lista de la compra

Una vez que tu menú semanal está perfecto, WeekFoods genera automáticamente una lista de la compra detallada con todos los ingredientes necesarios. ¡Para llevar en tu móvil!

Beneficio: Simplifica tus compras, evita olvidos y reduce el desperdicio de alimentos, haciendo tu vida más fácil y eficiente.

Estructura

Estructura de la base de datos y relaciones

La robustez de WeekFoods se basa en un diseño de base de datos relacional, gestionado eficientemente por el ORM de Django. Los modelos clave y sus interconexiones son:

Modelos principales:

  • User: El modelo de usuario estándar de Django para la autenticación.
  • UserWeekfoods: Extiende el modelo User, actuando como perfil de usuario y gestionando la relación de muchas a muchas con las recetas que cada usuario posee.
  • Recipe: Almacena los detalles de cada receta (nombre, elaboración, calorías, momento del día).
  • Ingredient: Contiene la información de cada ingrediente (nombre, tipo, precio).
  • WeeklyMenu: Relaciona un usuario con las 10 recetas específicas que componen su menú semanal generado o guardado.

Relaciones:

  • User ↔ UserWeekfoods: Relación One-to-One (un usuario tiene un perfil Weekfoods).
  • UserWeekfoods ↔ Recipe: Relación Many-to-Many (un usuario puede tener muchas recetas, y una receta puede pertenecer a muchos usuarios).
  • Recipe ↔ Ingredient: Relación Many-to-Many (una receta tiene muchos ingredientes, y un ingrediente puede estar en muchas recetas).
  • UserWeekfoods ↔ WeeklyMenu ↔ Recipe: Relación Many-to-Many a través de WeeklyMenu, que actúa como un modelo intermedio para almacenar el menú semanal específico de un usuario.

Desafíos y aprendizaje

Desafíos y aprendizajes

El desarrollo de WeekFoods presentó varios retos que fueron oportunidades de aprendizaje:

Retos superados:

  • Gestión de relaciones Many-to-Many: Implementar correctamente las relaciones entre usuarios y recetas, y entre recetas e ingredientes, asegurando la integridad de los datos.
  • Lógica de filtrado compleja: Desarrollar un algoritmo eficiente para generar menús aleatorios que cumplan múltiples criterios (calorías, presupuesto) y que se adapten a la disponibilidad de recetas del usuario.
  • Formularios dinámicos y validación: Manejar la creación y edición de recetas e ingredientes con formularios de Django, incluyendo validaciones personalizadas (ej. unicidad de ingredientes, formato de URLs de YouTube).

Aprendizajes:

  • Uso avanzado de vistas basadas en clases (CBV): Optimizar el código y la estructura de las vistas para manejar diferentes tipos de peticiones (GET, POST) y acciones (crear, eliminar, listar).
  • Gestión de sesiones y mensajes: Utilizar el framework de mensajes de Django para proporcionar feedback claro al usuario y la sesión para mantener estados temporales (como los checkboxes marcados).
  • Diseño responsivo y UX: La importancia de un diseño adaptable y una interfaz intuitiva para mejorar la experiencia del usuario, especialmente en la planificación del menú.

Tecnologías

Tecnologías detrás de WeekFoods

WeekFoods está construido con tecnologías modernas y robustas para ofrecerte la mejor experiencia:

Backend y base de datos:

  • Python: Lenguaje de programación principal, conocido por su legibilidad y versatilidad.
  • Django: Framework web de Python, elegido por su "baterías incluidas" que acelera el desarrollo, su ORM robusto y su panel de administración integrado.
  • PostgreSQL: Base de datos relacional potente y fiable, ideal para aplicaciones escalables.

Frontend y herramientas:

  • HTML5: Para la estructura y contenido de la web.
  • CSS3: Para el diseño y la estética, permitiendo una interfaz moderna y atractiva.
  • JavaScript: Interactividad del lado del cliente para el buscador de ingredientes.
  • Bootstrap: Framework CSS para un diseño responsivo rápido y consistente.
